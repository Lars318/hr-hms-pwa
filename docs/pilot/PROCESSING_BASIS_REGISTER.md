# Behandlingsgrunnlagsregister

> **Versjon:** 1.0 – Juni 2026  
> **⚠️ Merk:** Dette er et teknisk utkast. Behandlingsgrunnlag MÅ bekreftes av ledelse og/eller juridisk rådgiver før pilot.  
> **Hjemmel:** GDPR art. 30 krever register over behandlingsaktiviteter. GDPR art. 6 og 9 krever gyldig behandlingsgrunnlag.

---

## Register over behandlingsaktiviteter

### 1. Brukerprofil (ansattregistrering)

| Felt | Verdi |
|---|---|
| **Behandling** | Oppretting og vedlikehold av ansattprofil |
| **Personopplysninger** | Navn, e-post, stilling, tittel, avdeling, lokasjon(er), ansettelsesdato, status, rolle i systemet |
| **Formål** | Tilgangsstyring, identifikasjon, organisering av HMS-arbeid |
| **Foreslått behandlingsgrunnlag** | Art. 6(1)(b): Nødvendig for oppfyllelse av arbeidsavtale; Art. 6(1)(c): Rettslig forpliktelse (arbeidsgiver-dokumentasjonsplikt) |
| **Tilgang** | ADMIN, HR (full), MANAGER (egne ansatte), EMPLOYEE (egen profil) |
| **Lagringstid** | Aktiv ansettelse + 3 år etter avslutning |
| **Kommentar** | ⚠️ Bekreft med juridisk rådgiver |

---

### 2. Avviksmeldinger

| Felt | Verdi |
|---|---|
| **Behandling** | Registrering og oppfølging av HMS-avvik |
| **Personopplysninger** | Rapportørens navn, involvert persons navn (valgfritt), beskrivelse av hendelse |
| **Formål** | Oppfyllelse av internkontrollforskriften (IK-HMS), forebygging av gjentakelse |
| **Foreslått behandlingsgrunnlag** | Art. 6(1)(c): Rettslig forpliktelse (IK-HMS §5) |
| **Tilgang** | ADMIN, HR, HMS, MANAGER (egne lokasjoner/avdelinger), EMPLOYEE (egne avvik) |
| **Lagringstid** | 5 år fra registrering |
| **Kommentar** | Beskrivelse kan inneholde sensitive opplysninger om hendelse. Vurder om det bør varsles om at avvik kan inneholde personopplysninger. |

---

### 3. Risikovurderinger og tiltak

| Felt | Verdi |
|---|---|
| **Behandling** | HMS-risikovurderinger med risikopunkter og tiltak |
| **Personopplysninger** | Ansvarlig persons navn |
| **Formål** | Systematisk HMS-arbeid, oppfyllelse av Aml §3-1 |
| **Foreslått behandlingsgrunnlag** | Art. 6(1)(c): Rettslig forpliktelse (Aml §3-1, IK-HMS) |
| **Tilgang** | ADMIN, HR, HMS, MANAGER |
| **Lagringstid** | 5 år, eller til ny risikovurdering erstatter den |
| **Kommentar** | ⚠️ Bekreft med juridisk rådgiver |

---

### 4. Fravær (inkl. sykefravær)

| Felt | Verdi |
|---|---|
| **Behandling** | Registrering og godkjenning av fraværssøknader |
| **Personopplysninger** | Navn, fraværstype (inkl. sykefravær), periode, dager, status |
| **Særlige kategorier** | Sykefravær er helseopplysning (art. 9). Ingen diagnose lagres. |
| **Formål** | Lønnsgrunnlag, oppfyllelse av Aml §4-6 (oppfølging), NAV-rapportering |
| **Foreslått behandlingsgrunnlag** | Art. 6(1)(b)+(c): Arbeidsavtale og rettslig forpliktelse; Art. 9(2)(b): Nødvendig for sysselsetting (sykefravær) |
| **Tilgang** | ADMIN, HR (alle), MANAGER (egne ansatte), EMPLOYEE (egne søknader) |
| **Lagringstid** | 5 år |
| **Kommentar** | ⚠️ Sykefravær er særlig kategori – krever eget behandlingsgrunnlag. MÅ bekreftes juridisk. |

---

### 5. Overtid og timebank

| Felt | Verdi |
|---|---|
| **Behandling** | Registrering og godkjenning av overtid og avspasering |
| **Personopplysninger** | Navn, timer, type, dato, godkjenner |
| **Formål** | Lønnsgrunnlag, oppfyllelse av Aml §10-6 |
| **Foreslått behandlingsgrunnlag** | Art. 6(1)(b)+(c): Arbeidsavtale og rettslig forpliktelse |
| **Tilgang** | ADMIN, HR (alle), MANAGER (egne ansatte), EMPLOYEE (egne registreringer) |
| **Lagringstid** | 5 år |
| **Kommentar** | Appen er ikke komplett arbeidstidsregistrering. Full daglig registrering håndteres i separate systemer. |

---

### 6. Personalhåndbok – lesebekreftelser

| Felt | Verdi |
|---|---|
| **Behandling** | Registrering av hvem som har lest og bekreftet personalhåndbokvarianten |
| **Personopplysninger** | Navn, dato for bekreftelse, versjonsnummer |
| **Formål** | Dokumentere at ansatte har mottatt og lest gjeldende personalhåndbok |
| **Foreslått behandlingsgrunnlag** | Art. 6(1)(f): Berettiget interesse (dokumentasjonsbehov) |
| **Tilgang** | ADMIN, HR |
| **Lagringstid** | Aktiv ansettelse + 3 år |
| **Kommentar** | ⚠️ Bekreft at berettiget interesse er riktig grunnlag |

---

### 7. Dokumentlesing og lesebekreftelser

| Felt | Verdi |
|---|---|
| **Behandling** | Sporing av hvem som har bekreftet at de har lest interne dokumenter |
| **Personopplysninger** | Navn, dokumenttittel, dato for bekreftelse |
| **Formål** | Sikre at ansatte kjenner gjeldende rutiner og prosedyrer |
| **Foreslått behandlingsgrunnlag** | Art. 6(1)(f): Berettiget interesse (HMS-dokumentasjonsplikt) |
| **Tilgang** | ADMIN, HR (statistikk), EMPLOYEE (egne bekreftelser) |
| **Lagringstid** | 3 år etter siste versjon |
| **Kommentar** | ⚠️ Bekreft behandlingsgrunnlag |

---

### 8. Varsler og notifikasjoner

| Felt | Verdi |
|---|---|
| **Behandling** | Sending og sporing av interne varsler |
| **Personopplysninger** | Mottakers navn og e-post, varselinnhold (kan inneholde referanse til hendelse/person) |
| **Formål** | Operasjonell kommunikasjon, HMS-varsling |
| **Foreslått behandlingsgrunnlag** | Art. 6(1)(f): Berettiget interesse (driftsformål) |
| **Tilgang** | ADMIN, HR (alle), mottaker (egne varsler) |
| **Lagringstid** | 1 år |
| **Kommentar** | Varselinnhold kan inneholde personopplysninger indirekte |

---

### 9. Audit-logg

| Felt | Verdi |
|---|---|
| **Behandling** | Logg over vesentlige handlinger i systemet |
| **Personopplysninger** | Brukerens ID/navn, handling, tidspunkt, berørt entitet |
| **Formål** | Sikkerhet, sporbarhet, etterforskning ved hendelser |
| **Foreslått behandlingsgrunnlag** | Art. 6(1)(f): Berettiget interesse (informasjonssikkerhet) |
| **Tilgang** | ADMIN |
| **Lagringstid** | 2 år |
| **Kommentar** | Logg bør ikke inneholde mer enn nødvendig |

---

### 10. Push-varsling-abonnementer

| Felt | Verdi |
|---|---|
| **Behandling** | Lagring av enhets-token for push-varsler |
| **Personopplysninger** | Enhets-token koblet til bruker |
| **Formål** | Sende push-varsler til ansattes enheter |
| **Foreslått behandlingsgrunnlag** | Art. 6(1)(a): Samtykke (ansatt aktiverer push i appen) |
| **Tilgang** | System (automatisk), ADMIN |
| **Lagringstid** | Til ansatt deaktiverer, avslutter, eller ansettelsen opphører |
| **Kommentar** | Samtykke kan trekkes tilbake når som helst i appen |

---

### 11. Rapporter og CSV-eksport

| Felt | Verdi |
|---|---|
| **Behandling** | Generering av rapporter med personopplysninger |
| **Personopplysninger** | Navn, avvik, fravær, overtid m.m. – avhengig av rapporttype |
| **Formål** | HR-administrasjon, HMS-statistikk, lønnsgrunnlag |
| **Foreslått behandlingsgrunnlag** | Samme som for underliggende data (art. 6(1)(b)/(c)/(f)) |
| **Tilgang** | ADMIN, HR, MANAGER (begrenset) |
| **Lagringstid** | Eksporterte rapporter håndteres av mottaker – ikke lagret i appen |
| **Kommentar** | Eksporterte filer bør behandles i henhold til intern dataklassifisering |
